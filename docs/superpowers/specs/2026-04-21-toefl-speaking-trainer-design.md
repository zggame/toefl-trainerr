# TOEFL Speaking Trainer — App Design Spec

**Date:** 2026-04-21
**Status:** Approved
**Priority order of build:** Practice loop → Retry → Auth → (Instructor dashboard out of scope)

---

## 1. Concept & Vision

A TOEFL speaking practice app for college and high school students. The app simulates the real TOEFL speaking test (2023+ format: 11 tasks, audio-driven prompts, ~8 minutes) and gives learners AI-powered scoring with evidence-based feedback and a targeted retry loop. Practice-first, test-ready.

---

## 2. Target Users

- College and high school students (ages 16–24) preparing for TOEFL iBT
- Self-study learners practicing independently
- Sign-in required via Google OAuth (Supabase Auth)

---

## 3. Design System

- **Style:** Claymorphism — soft 3D, thick borders (3px), double shadows, rounded (20px), pill-shaped buttons
- **Audience mood:** Playful, friendly, energetic, modern — not corporate, not childish
- **Typography:** Baloo 2 (headings), Comic Neue (body)
- **Colors:**
  - Primary: `#4F46E5` (indigo)
  - CTA: `#22C55E` (green, with 3D button effect)
  - Background: `#EEF2FF`
  - Text: `#312E81`
- **Shadows:** Claymorphism depth levels (sm/md/lg/xl) with double-shadow syntax
- **Icons:** Lucide React — no emojis as icons
- **Pre-delivery checklist:** cursor-pointer, smooth transitions, 4.5:1 contrast, responsive

## 4. Architecture

### Tech Stack
- **Framework:** Next.js 16 (App Router)
- **Styling:** Tailwind CSS v4 + Claymorphism design system
- **Database:** Supabase (existing project, new tables)
- **AI:** Google Gemini (audio transcription + scoring, audio input supported directly)
- **Auth:** Supabase Auth (Google OAuth)

### API Routes

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/toefl/tasks` | GET | Fetch a task (random by category or specific) |
| `/api/toefl/score` | POST | Send audio base64 → Gemini → return transcript + score + feedback; background save audio to Supabase Storage |
| `/api/toefl/attempts` | GET | Fetch attempt history for authenticated user |
| `/api/toefl/attempts/[id]` | GET | Fetch single attempt (with audio playback URL) |
| `/api/toefl/profile` | GET/PUT | Get or update learner profile |

### Data Flow

```
Learner records answer
        ↓
Send audio base64 → Gemini
        ↓
Receive: { transcript, delivery, languageUse, topicDev, overallScore, errors, suggestion }
        ↓
Background: upload audio to Supabase Storage (async)
        ↓
Save to toefl_attempts (with storage URL)
        ↓
Return to learner (fast — storage happens in background)
```

No separate upload step. One API call handles score + async save.

## 6. Prompt Library

### Strategy
- Audio prompts generated by **Gemini TTS** (Option B from research)
- A setup script generates a prompt bank: creates audio files, uploads to Supabase Storage, seeds `toefl_tasks` table
- Later: swap in human-recorded audio for more authenticity
- Prompt bank organized by category (listen_repeat / interview), difficulty, topic domain

### Audio Delivery
- Audio prompts played via browser audio player from Supabase Storage
- Plays automatically once when task loads (like the real test)
- Replay button available during prep time
- No reading passages in the new format — audio-only input

### Show Text Feature
- Each task has a `transcript` field
- In guided mode: Show Text toggle reveals the audio transcript
- In simulation mode: hidden (exam-accurate)
- Future: assistant practice mode may show text before audio

## 8. Guided vs. Simulation Mode

| Feature | Simulation | Guided |
|---------|-----------|--------|
| Audio replay | Once only (like test) | Unrestricted |
| Show Text | Hidden | Available |
| Scaffolding | None | Prep checklist, response hints |
| Score display | Score only | Full breakdown + coaching |
| Retry options | Full retake | Full / Targeted / Sentence |

Default: Guided for new learners. Simulation toggle available on home screen.

### Scaffolding Tools (guided mode)
- Prep checklist shown during prep countdown
- Response hint (e.g., structure your answer with 2 points)
- Show Text toggle after audio plays

## 9. Scoring Algorithm

Gemini receives audio (base64) + task context + rubric, returns structured result:

```typescript
{
  delivery: { score: 3, evidence: string, tip: string },
  languageUse: { score: 3, evidence: string, tip: string },
  topicDev: { score: 4, evidence: string, tip: string },
  overallScore: 3.3,    // weighted average
  errors: string[],     // ['article', 'vague_detail']
  suggestion: string    // coaching tip
}
```

Weights: Delivery 30%, Language Use 30%, Topic Development 40%.
Only Gemini — no separate audio analysis step.

## 10. Retry Loop

### Full Retake
- Replay audio prompt, full prep + record cycle
- New attempt saved, previous attempt linked via `previous_attempt_id`
- Side-by-side comparison shown in review

### Targeted Retry
1. Show transcript with segment highlights (introduction, main point 1, main point 2, conclusion)
2. Learner taps a weak segment
3. Re-record just that segment (~10s)
4. New segment scored + compared to old
5. Option to re-record another segment or finish

### Sentence-Level Retry
- After selecting a segment, drill down to specific sentences
- Re-record just that sentence (~3–5s)
- Audio comparison playback

## 11. Auth

- **Required:** Google OAuth via Supabase Auth before any practice
- Flow: `/toefl` → redirect to sign-in → Continue with Google → create/update `toefl_profiles` → show dashboard
- No anonymous practice
- Same `auth.users` as existing smart-interview Supabase project

- Instructor/coach dashboard
- Anonymous practice
- Video prompts (audio only)
- Cross-exam support (IELTS, PTE) — TOEFL first only
- Full test simulation (11 tasks in one session) — MVP does single tasks
