# TOEFL Speaking Research — Task 7: Instructional Scaffolding Tools

**Last updated:** 2026-04-20

---

## 1. Overview

Scaffolding tools help learners prepare to speak *before* they record, not only receive feedback *after*. The key design question is how much scaffolding is useful before it becomes a crutch that prevents independent speaking ability from developing.

---

## 2. The Scaffolding-to-Independence Spectrum

| Level | What the app provides | When to use |
|-------|----------------------|-------------|
| 0 — None | Prompt only, timer, record | Simulation mode, advanced learners |
| 1 — Light | Time allocation reminder, topic keywords | Early simulation practice |
| 2 — Medium | Outline template, prep time countdown | Guided practice for beginners |
| 3 — Heavy | Full response framework, example answer | First encounter with a task type |
| 4 — Maximum | Complete script to read aloud | Diagnostic or drill mode only |

**Design principle:** Start at level 2–3 for new learners, progressively remove scaffolding as score improves. The Hybrid Workflow (Workflow D) naturally handles this transition.

## 3. Specific Scaffolding Tools

### 3.1 Quick Response Templates (Independent Tasks)

**Purpose:** Help learners structure a response quickly without starting from blank page.

**Template for Task 1 (Personal Experience):**
```
My [topic] is [specific thing].
There are [two/three] reasons why I feel this way.
First, [main point with specific detail].
Second, [main point with specific detail]. 
In conclusion, [restate opinion in new words].
```

**Template for Task 1 (Opinion/Choice):**
```
I strongly believe that [opinion/choice].
The main reason is [point with example].
Another reason is [point with example].
For these reasons, [restate position].
```

**Usage note:** Templates should be shown *before* prep time starts, then hidden during recording. They help plan, not memorize.

### 3.2 Outline Frame (Integrated Tasks)

**Purpose:** Help learners organize reading and listening information before speaking.

**Outline for Task 2 (Campus):**
```
Reading proposes: [main change/policy]
Speaker's opinion: [agrees/disagrees]
Reason 1: [specific point from listening]
Reason 2: [specific point from listening]
Conclusion: [speaker's final stance]
```

**Outline for Task 3 (Academic):**
```
Main concept: [key term or idea from lecture]
Key point 1: [first major idea with example]
Key point 2: [second major idea with example]
Conclusion: [how this concept applies or why it matters]
```

### 3.3 Keyword Board

**Purpose:** Help learners note key terms from reading/listening that they might forget during speaking.

**Interface:** A simple text field shown during prep time where learners can type 5–10 keywords they want to remember. These become visible during the prep countdown and disappear when recording starts.

### 3.4 Mind-Map Prompt

**Purpose:** For independent tasks, help learners brainstorm relevant vocabulary and ideas.

**Interface:** A single text prompt that appears during prep time:
> Quick brainstorm: Write 3 specific words related to this topic, 1 example of something, and 1 reason why.

### 3.5 Integrated-Task Note-Taking Helper

**Purpose:** Structured note-taking field for integrated tasks (reading + listening).

**Interface:**
```
READING NOTES:
- Main point:
- 2 supporting details:

LISTENING NOTES:
- Main point:
- 2 supporting details:

MY RESPONSE PREP:
- My position:
- 2 key reasons:
```

### 3.6 Time-Management Cue

**Purpose:** Help learners allocate prep time wisely.

**Interface:** During prep countdown, show a simple progress bar:
```
[====----] 15s left
Time to plan, not write!
Key point: ____________
```

### 3.7 Source-Detail Capture Prompts

**Purpose:** For integrated tasks, prompt learners to capture specific source details they might otherwise miss.

**Interface:** During prep time, show:
```
Don't forget to mention:
□ A specific detail from the reading
□ The speaker's exact opinion
□ One specific example from the lecture
```

## 4. Scaffolding Default Settings

| User Level | Default Scaffolding | Configurable |
|------------|--------------------|--------------|
| First-time / score < 3 | Templates + outline + keywords | Yes — can turn off |
| Intermediate / score 3–4 | Light templates + keywords only | Yes |
| Advanced / score >= 4 | Prep timer only | Yes |
| Simulation mode | None | No — full simulation |

---

## 5. Warning: Scaffolding as Crutch

**The risk:** If learners rely on templates too heavily, they:
- Sound robotic and formulaic
- May score lower on Topic Development if the template replaces genuine content
- May struggle when the real test doesn't match their template
- Lose the ability to think on their feet

**Mitigation strategies:**
1. **Fade out:** Gradually remove scaffolding as scores improve
2. **Variety:** Use different template styles so learners don't over-rely on one structure
3. **Time pressure:** Even in guided mode, keep prep time tight (15–30s) to prevent over-reliance
4. **Reflection prompt:** After using a template, ask: Could you say this without the template?
5. **Random tasks:** Don't always use the same template — vary prompts so structure becomes natural
